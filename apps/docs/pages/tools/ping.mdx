# Ping 测试

Ping 测试工具用于测试目标主机的延迟和丢包率，基于 ICMP 协议。

## 功能特点

- 多节点并发测试
- 延迟统计（最小值、最大值、平均值）
- 丢包率计算
- 往返时间（RTT）测量
- 持续监控模式

## 使用方法

### 基础测试

1. 输入目标主机（域名或 IP 地址）
2. 选择测试节点
3. 设置 Ping 次数（默认 4 次）
4. 点击"开始测试"

### 高级选项

- **Ping 次数**：发送的 ICMP 包数量
- **包大小**：ICMP 包的大小（字节）
- **超时时间**：每个 Ping 的超时时间
- **间隔时间**：两次 Ping 之间的间隔

## 结果解读

测试结果包括：

- **最小延迟**：所有 Ping 中的最小 RTT
- **最大延迟**：所有 Ping 中的最大 RTT
- **平均延迟**：所有成功 Ping 的平均 RTT
- **丢包率**：丢失的包占总包数的百分比
- **抖动**：延迟的标准差，反映网络稳定性

## 延迟指标参考

- **< 30ms**：优秀，适合实时游戏和视频通话
- **30-50ms**：良好，日常使用流畅
- **50-100ms**：一般，可能有轻微延迟感
- **100-200ms**：较慢，影响实时交互
- **> 200ms**：很慢，需要优化

## 丢包率参考

- **0%**：完美
- **< 1%**：优秀
- **1-5%**：可接受
- **> 5%**：需要关注网络质量
