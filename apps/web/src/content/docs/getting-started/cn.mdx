# 快速开始

## 什么是 idcd？

idcd 是一个多节点网络诊断工具平台，提供全球分布式拨测服务。通过 idcd，你可以：

- 从全球多个节点测试你的服务可达性
- 获取真实的网络延迟和性能数据
- 快速定位网络问题的地理位置
- 生成可信的网络诊断证据

## 节点分布

idcd 在全球多个地区部署了测试节点：

- **亚洲**：中国大陆（北京、上海、广州）、香港、日本、新加坡
- **北美**：美国东西海岸
- **欧洲**：英国、德国、法国

## 快速使用

1. 访问 [在线工具](/tools) 页面
2. 选择需要的工具（如 HTTP 拨测、Ping 测试）
3. 输入目标域名或 IP
4. 点击开始，等待全球节点返回结果

## API 接入

idcd 提供 REST API 和 MCP Server，方便程序化接入：

- [MCP Server](/docs/mcp) — 让 Claude、Cursor 等 AI 直接调用拨测能力
- REST API — 参考 API 文档（即将发布）

## 步骤 1：创建你的第一个监控

1. 登录后进入 [监控列表](/app/monitors)，点击右上角"创建监控"按钮
2. 选择监控类型：
   - **HTTP/HTTPS**：检测 API 或网站可用性（最常用）
   - **Ping**：检测服务器网络可达性
   - **TCP**：检测指定端口连通性
   - **SSL 到期**：监控 HTTPS 证书即将到期告警
3. 填写目标地址，例如 `https://api.example.com/health`
4. 设置检测频率（推荐 **5 分钟**，高可用业务可选 1 分钟）
5. 高级配置（可选）：
   - 断言状态码：期望返回 `200`
   - 关键字匹配：响应体必须包含指定字符串
   - 超时时间：单次拨测最长等待时间（默认 10s）

## 步骤 2：配置告警通道

1. 进入 [告警中心](/app/alerts) → 告警通道标签页
2. 点击"添加通道"，选择通知方式：
   - **Email**：直接发送到你的邮箱
   - **企业微信**：通过 Webhook 机器人推送
   - **飞书**：通过飞书自定义机器人推送
   - **Webhook**：自定义 HTTP 回调，接入任意系统
3. 点击通道右侧的"测试"按钮，确认通道可正常接收消息
4. 创建告警策略：绑定监控项 + 通知通道，并设置告警延迟（0 = 立即告警，建议设 2-3 分钟避免误报）

## 步骤 3：查看监控数据

- **仪表盘**：总体健康状态概览，置顶关键监控项
- **监控详情**：24 小时延迟趋势图、平均响应时间、在线率统计
- **故障记录**：查看历史故障时间线，生成复盘报告
- **SLA 月报**：在 [报告中心](/app/reports) 下载 PDF 或 CSV 格式的月度可用性报告

## API 接入（开发者）

1. 在 [设置 → API Keys](/app/settings/api-keys) 页面创建你的 API Key
2. 在请求头中携带 Key：

   `Authorization: Bearer <your-key>`

3. 查阅完整接口文档：[API 参考](/docs/mcp)

## 常见问题

**Q：监控显示 DOWN 但实际服务正常？**

检查节点选择范围，可能是特定地区网络波动导致。可尝试增大超时时间，或切换到距离目标服务更近的节点。

**Q：收到太多告警误报怎么办？**

在告警策略中调整"延迟分钟数"，例如设为 3 分钟，即连续失败超过 3 分钟才触发通知，可有效过滤短暂抖动。

**Q：如何对外共享服务状态？**

在 [状态页](/app/status-pages) 中创建公开状态页，可自定义域名，适合对用户或客户公开服务健康状况。

**Q：多人团队如何协作？**

在 [设置 → 团队成员](/app/settings) 中邀请成员，不同角色拥有不同的查看与操作权限。
