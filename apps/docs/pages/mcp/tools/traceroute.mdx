# traceroute Tool

追踪数据包到目标主机的网络路径，逐跳显示 IP、延迟、ASN 和 ISP 信息。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `target` | string | ✓ | — | 目标主机名或 IP 地址 |
| `max_hops` | integer | — | 30 | 最大跳数限制 |

## 示例

**请求**：

```json
{"target": "8.8.8.8", "max_hops": 15}
```

**输出**：

```
TRACEROUTE 8.8.8.8
 1. 192.168.1.1          0.5ms
 2. 10.0.0.1             2.3ms
 3. 203.0.113.1          8.7ms (AS4837, ChinaUnicom)
 4. 202.97.58.34        12.1ms (AS4837, ChinaUnicom)
 5. 64.233.175.69       28.4ms (AS15169, Google LLC)
 6. 8.8.8.8             30.2ms (AS15169, Google LLC)
```

**仅主机名**：

```json
{"target": "github.com"}
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 返回字段说明

每一跳显示：

| 字段 | 说明 |
|------|------|
| 跳数 | TTL 值，从 1 开始递增 |
| IP | 该跳路由器的 IP 地址 |
| 延迟 | 到达该跳的往返时间（毫秒） |
| ASN | 自治系统号（如 AS15169） |
| ISP | 运营商或组织名称 |

## 常见使用场景

- 定位高延迟发生在哪一跳
- 判断流量经过哪些运营商
- 排查跨境链路质量问题
- 验证 CDN 回源路径是否正常

## API 端点

内部调用 `POST /v1/probe/traceroute`，认证方式为 `Authorization: Bearer <key>`。
