# diagnose Tool

一键执行全面网络诊断，内部组合调用 DNS 解析、HTTP 检测、SSL 证书检查和 Ping 测试，返回综合诊断报告。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `target` | string | ✓ | — | 目标主机名或 URL（支持 `https://example.com` 或 `example.com`） |

## 示例

**域名请求**：

```json
{"target": "github.com"}
```

**输出**：

```
DIAGNOSE github.com
[✓] DNS: 3 A records
[✓] HTTP: 200 OK (145ms)
[✓] SSL: valid, expires 2025-03-14
[✓] Ping: avg 32ms, 0% loss
```

**URL 请求**：

```json
{"target": "https://api.example.com/health"}
```

**输出**（部分检测失败示例）：

```
DIAGNOSE https://api.example.com/health
[✓] DNS: 1 A records
[✗] HTTP: API error 503
[✓] SSL: valid, expires 2026-01-01
[✓] Ping: avg 55ms, 0% loss
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 诊断步骤说明

| 步骤 | 说明 |
|------|------|
| DNS | 查询目标的 A 记录，确认域名可以解析 |
| HTTP | 对 `https://target` 发起 GET 请求，检测响应状态和延迟 |
| SSL | 检查 HTTPS 证书有效性和到期时间 |
| Ping | 发送 3 个 ICMP 包，测量平均延迟和丢包率 |

状态前缀：`[✓]` 表示检测通过，`[✗]` 表示检测失败或出错。

## 适用场景

- 网站上线前的快速健康验证
- 告警触发后的一键定位
- 排查"网站无法访问"类问题
- 对比不同域名的全面网络状态

## 与单工具的关系

`diagnose` 是以下工具的组合快捷方式，不支持自定义各子检测的参数。如需精细控制，请分别调用对应工具：

- [ping](/mcp/tools/ping)
- [http](/mcp/tools/http)
- [ssl](/mcp/tools/ssl)
- [dns](/mcp/tools/dns)

## API 端点

内部依次调用 `GET /v1/info/dns`、`POST /v1/probe/http`、`GET /v1/info/ssl`、`POST /v1/probe/ping`，认证方式为 `Authorization: Bearer <key>`。
