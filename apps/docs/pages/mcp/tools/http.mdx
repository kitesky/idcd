# http Tool

检测 HTTP/HTTPS 端点的可达性、响应状态码、延迟和 TLS 信息。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `url` | string | ✓ | — | 目标 URL，必须包含 `http://` 或 `https://` 前缀 |
| `method` | string | — | `GET` | HTTP 方法，可选值：`GET`、`HEAD`、`POST` |

## 示例

**请求**：

```json
{"url": "https://example.com", "method": "GET"}
```

**输出**：

```
GET https://example.com
状态: 200 OK | 延迟: 123ms | TLS: TLS 1.3
响应头: Content-Type: text/html; charset=UTF-8
```

**HEAD 请求**：

```json
{"url": "https://api.example.com/health", "method": "HEAD"}
```

**输出**：

```
HEAD https://api.example.com/health
状态: 200 OK | 延迟: 45ms | TLS: TLS 1.3
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 返回字段说明

| 字段 | 说明 |
|------|------|
| 状态 | HTTP 状态码和状态文本（如 `200 OK`、`404 Not Found`） |
| 延迟 | 完整请求响应时间（毫秒） |
| TLS | TLS 版本，仅 HTTPS 时显示 |
| 响应头 | `Content-Type` 响应头，如有则显示 |

## 常见使用场景

- 确认网站是否正常返回 200
- 检测 API 健康检查端点
- 验证 HTTPS 证书配置（TLS 版本）
- 测量全球节点到目标 URL 的响应时间

## API 端点

内部调用 `POST /v1/probe/http`，认证方式为 `Authorization: Bearer <key>`。
