# dns Tool

对域名执行 DNS 解析查询，支持多种记录类型，返回解析结果和 TTL 值。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `domain` | string | ✓ | — | 要解析的域名 |
| `type` | string | — | `A` | DNS 记录类型，可选：`A`、`AAAA`、`CNAME`、`MX`、`TXT`、`NS` |

## 示例

**查询 A 记录**：

```json
{"domain": "google.com", "type": "A"}
```

**输出**：

```
DNS google.com (A):
A: 142.250.66.46 (TTL: 300)
A: 142.250.66.14 (TTL: 300)
```

**查询 MX 记录**：

```json
{"domain": "gmail.com", "type": "MX"}
```

**输出**：

```
DNS gmail.com (MX):
MX: alt1.gmail-smtp-in.l.google.com (TTL: 3600)
MX: alt2.gmail-smtp-in.l.google.com (TTL: 3600)
```

**查询 TXT 记录**（常用于 SPF/DKIM 验证）：

```json
{"domain": "example.com", "type": "TXT"}
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 记录类型说明

| 类型 | 用途 |
|------|------|
| `A` | IPv4 地址（最常用） |
| `AAAA` | IPv6 地址 |
| `CNAME` | 域名别名（指向另一个域名） |
| `MX` | 邮件服务器 |
| `TXT` | 文本记录（SPF、DKIM、域名验证等） |
| `NS` | 权威名称服务器 |

## API 端点

内部调用 `GET /v1/info/dns?q={domain}&type={type}`，认证方式为 `Authorization: Bearer <key>`。
