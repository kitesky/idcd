# ping Tool

对目标主机发起多节点 ICMP Ping 检测，返回各节点延迟和丢包率。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `target` | string | ✓ | — | 目标主机名或 IP 地址 |
| `count` | integer | — | 3 | 每个节点发送的 ICMP 包数量 |

## 示例

**请求**：

```json
{"target": "google.com", "count": 3}
```

**输出**：

```
PING google.com
节点: Tokyo JP — 32ms ✓
节点: Singapore SG — 45ms ✓
节点: Los Angeles US — 152ms ✓
平均: 76ms | 丢包: 0%
```

**IP 地址请求**：

```json
{"target": "8.8.8.8", "count": 5}
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 返回字段说明

| 字段 | 说明 |
|------|------|
| 节点行 | `节点名 国家代码 — 平均延迟ms 状态符号` |
| 平均 | 所有节点的综合平均延迟 |
| 丢包 | 综合丢包率百分比 |

状态符号：`✓` 表示节点可达，`✗` 表示节点不可达。

## 延迟参考

| 延迟范围 | 评价 |
|----------|------|
| < 30ms | 优秀 |
| 30–80ms | 良好 |
| 80–150ms | 一般 |
| > 150ms | 较慢 |

## API 端点

内部调用 `POST /v1/probe/ping`，认证方式为 `Authorization: Bearer <key>`。
