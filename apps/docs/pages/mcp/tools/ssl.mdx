# ssl Tool

检测域名的 SSL/TLS 证书有效性、到期时间、颁发机构和协议版本。

## 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `host` | string | ✓ | — | 要检测证书的主机名（不含 `https://`） |

## 示例

**请求**：

```json
{"host": "github.com"}
```

**输出**：

```
SSL/TLS for github.com:
证书: github.com | 颁发者: DigiCert TLS RSA SHA256 2020 CA1
有效期: 2025-03-14 (还有 295 天)
协议: TLS 1.3
```

**请求**（即将到期的证书）：

```json
{"host": "expired.badssl.com"}
```

**输出**：

```
SSL/TLS for expired.badssl.com:
证书: *.badssl.com | 颁发者: COMODO RSA Domain Validation Secure Server CA
有效期: 2015-04-12 (还有 -3691 天)
协议: TLS 1.2
```

## 无 API Key 时的输出

```
⚠ 需要 API key，请设置 IDCD_API_KEY 环境变量
```

## 返回字段说明

| 字段 | 说明 |
|------|------|
| 证书 | 证书的 Subject（通常为域名或通配符） |
| 颁发者 | 签发该证书的 CA 机构 |
| 有效期 | 证书到期日期（YYYY-MM-DD 格式） |
| 还有 N 天 | 距到期的剩余天数，负数表示已过期 |
| 协议 | TLS 版本（TLS 1.2 / TLS 1.3） |

## 证书到期预警参考

| 剩余天数 | 建议操作 |
|----------|----------|
| > 30 天 | 正常，无需操作 |
| 7–30 天 | 安排更新计划 |
| < 7 天 | 立即更新 |
| < 0 天 | 证书已过期，HTTPS 访问将报错 |

## API 端点

内部调用 `GET /v1/info/ssl?q={host}`，认证方式为 `Authorization: Bearer <key>`。
